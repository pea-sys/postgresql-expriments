pg_read_all_settingsロールの動作ドウサ確認カクニン 　 　
postgresqlには次ツギのデフォルトロールが用意ヨウイされています
今回コンカイはpg_read_all_settingsについて動作ドウサ確認カクニンします
ロール 可能なアクセス
pg_read_all_settings 通常スーパーユーザのみが読み取れる、全ての設定変数を読み取る
pg_read_all_stats 通常スーパーユーザのみが読み取れる、すべてのpg_stat_*ビューを読み取り、
各種の統計関連のエクステンションを使用する
pg_stat_scan_tables 潜在的に長時間、テーブルのACCESS SHAREロックを取得する可能性がある監視機能を実行する
pg_signal_backend 他のバックエンドにシグナルを送信する（例：問い合わせのキャンセル、プロセスの終了）
pg_monitor 各種の監視ビューや機能を読み取り/実行する。
このロールは、pg_read_all_settings、pg_read_all_statsおよびpg_stat_scan_tablesのメンバである。
■pg_read_all_settings
まず、設定セッテイ変数ヘンスウとは何ナニかという話ハナシになりますが、pg_settingビューで取トれる値アタイのことと推測スイソク。
スーパーユーザーとそれ以外イガイで読ヨみ取トれる設定セッテイ変数ヘンスウの違チガいがあるか確認カクニンします。
1)一般イッパンユーザー作成サクセイ
　 CREATE ROLE general_user PASSWORD 'g' LOGIN; 　 　
2)DB作成サクセイ
　 CREATE DATABASE testdb WITH TEMPLATE = template0 ENCODING = 'UTF8' LC_COLLATE = 'Japanese_Japan.932' LC_CTYPE = 'Japanese_Japan.932';
3)一般イッパンユーザーが参照サンショウ可能カノウな設定セッテイテーブルの作成サクセイ
　 psql -U general_user -p5432 -d testdb -c "create table general_setting as  select * from pg_settings;" 　 　
　 ------------------------------- 　 　
　 SELECT 308 　 　 　
　 ------------------------------- 　 　
4)スーパーユーザーが参照サンショウ可能カノウな設定セッテイテーブルの作成サクセイ
　 psql -U postgres -p5432 -d testdb -c "create table super_setting as  select * from pg_settings;" 　 　
　 ------------------------------- 　 　
　 SELECT 329 　 　 　
　 ------------------------------- 　 　
5)設定セッテイ差分サブンを確認カクニンする

6)一般イッパンユーザーにpg_read_all_settingsの権限ケンゲンを与アタえます
　 psql -U postgres -p5432 -d testdb -a -c "GRANT pg_read_all_settings TO general_user;" 　 　
7)再度サイド、一般イッパンユーザーで設定セッテイを参照サンショウします
　 psql -U general_user -p5432 -d testdb -c "select count(name) from pg_settings;" 　 　
　  count 　 　 　
　 ------- 　 　 　
　 329 　 　 　
特権トッケンユーザーと同オナじ設定セッテイ数スウを参照サンショウできています。
特権トッケンユーザーが見ミれない設定セッテイはないはずなので、これで大丈夫ダイジョウブでしょう。
